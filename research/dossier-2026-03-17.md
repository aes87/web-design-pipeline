## Part I: AI Web Design Pipelines

### 1.1 The Landscape: Who's Doing What

**No one has built what we're building.** After exhaustive search, no public project connects YAML design briefs → W3C DTCG tokens → CSS custom properties → LLM-driven HTML/CSS/JS → Playwright+axe-core validation → screenshot-based iteration as an end-to-end closed loop. Individual pieces exist; the integration is novel.

#### Closest Competitors

| Project / Approach | What It Does | Strength | Gap vs. Our Pipeline |
|---|---|---|---|
| **v0 (Vercel)** | Natural language → React/Next.js components | Best UI quality of app builders | React-only, no animation choreography, no validation loop, no design tokens |
| **Bolt.new** | Prompt → full-stack app (8+ frameworks) | Framework flexibility, speed | Context loss at 15+ components, no award-quality output |
| **Lovable.dev** | Chat → React+TS+shadcn app | Full-stack from one interface | Single framework, basic design quality |
| **Cursor + Playwright MCP** (egghead.io) | Screenshot → diff → iterate loop | Closest to our visual iteration model | Mixed results, no structured briefs, no token system |
| **Codrops "Design Constitution"** | .cursorrules forbid raw values, force token usage | Achieved ~80 Lighthouse with GSAP+Three.js | Manual workflow, no automation, Cursor-specific |
| **Leon Furze** (Claude Code) | Content folder → Astro site in 20 min | Parallel agent validation teams | No design token pipeline, no animation patterns |
| **Raduan** (Claude Code) | 300-line design system doc → landing page | Demonstrated that micro-interactions separate AI slop from polish | Manual, no iteration loop, no validation automation |

#### The "70% Problem"

All design-to-code AI tools share a ceiling: they accelerate the first ~70% (scaffolding, basic layout) but struggle with the remaining 30% that defines production quality — custom animations, scroll-driven narratives, performance optimization, accessibility compliance, and design "soul." (Addy Osmani, multiple tool comparisons)

#### Key Architectural Pattern: Generate-Validate-Iterate

The most successful pipelines share this core:
```
Brief/Spec → Generate Code → Validate (multi-signal) → Review → Edit → Repeat
                                  │
                                  ├── HTML validation
                                  ├── Accessibility (axe-core)
                                  ├── Visual regression (Playwright + Pixelmatch)
                                  ├── Performance (Lighthouse)
                                  └── Screenshots (multi-viewport)
```

**Our pipeline already implements this.** The differentiation is in the structured intake (tiered briefs), the token-constrained styling, and the prose pattern library.

#### LLM Design Quality Benchmark

A practical test of 5 LLMs redesigning a landing page scored:
- Gemini 2.5: 85/100
- o1 Pro High: 80/100
- Claude 3.7 Sonnet: 75/100
- DeepSeek R1: 65/100
- Grok 3: 40/100

**Critical finding:** All models produced "an average of all designs worldwide" rather than distinctive solutions. The Design Constitution pattern (constraining AI with strict token rules) is the proven mitigation.

## Part II: Animation, Motion & Creative Coding

### 2.1 GSAP: Now 100% Free (v3.13, May 2025)

Webflow acquired GreenSock in October 2024. As of GSAP v3.13, **every plugin is free** including commercial use:

**Newly free plugins we should document and use:**
- **SplitText** — Complete rewrite: `autoSplit: true` (ResizeObserver + font load re-split), `deepSlice` (handles nested `<strong>`/`<em>`), `Intl.Segmenter()` for emoji/international text, built-in masking for reveals, automatic ARIA attributes. ~7KB, 50% smaller.
- **MorphSVG** — SVG shape morphing
- **DrawSVG** — Animated SVG stroke drawing
- **ScrollSmoother** — Smooth scroll with parallax (alternative to Lenis within GSAP ecosystem)
- **Flip** — FLIP-technique layout animations
- **Inertia** — Momentum/velocity-based animations
- **Physics2D / PhysicsProps** — Physics simulation
- **ScrambleText** — Text scramble effects
- **GSDevTools** — Visual animation debugging/scrubbing
- **CustomEase / CustomWiggle / CustomBounce** — Advanced easing

**GSAP remains dominant.** ~888K websites tracked by Wappalyzer. No challenger has emerged for complex, timeline-based web animations. Awwwards SOTD winners through Feb 2026 continue to use GSAP.

### 2.3 CSS Scroll-Driven Animations — Production Ready

**Browser support (March 2026):**

| Feature | Chrome | Edge | Safari | Firefox | Global |
|---|---|---|---|---|---|
| `animation-timeline: scroll()` | 115+ | 115+ | 26+ | Flag only | ~83% |
| `animation-timeline: view()` | 115+ | 115+ | 26+ | Flag only | ~83% |

**Performance:** Run on compositor thread — never block main thread, no layout recalculations, smooth 60fps during heavy JS. Zero JavaScript overhead.

**Strategy:** Use CSS scroll-driven for simple parallax/reveal effects. Layer GSAP ScrollTrigger for complex orchestrations. Feature-detect with `@supports (animation-timeline: scroll())`.

Polyfill available: `flackr/scroll-timeline` for Firefox fallback.

### 2.4 View Transitions API — ~90% Global Support

**Same-document (SPA):** Baseline Newly Available (Oct 2025) — Chrome 111+, Edge 111+, Safari 18+, Firefox 144+.

**Cross-document (MPA):** Chrome 126+, Edge 126+, Safari 18.2+. **Firefox: not yet.**

**New 2025 features:**
- `view-transition-name: match-element` (Chrome 137+) — auto-naming, no manual element IDs
- Nested `::view-transition-group` (Chrome 140) — clipping + 3D transforms during transitions
- Scoped view transitions (Chrome 140) — `element.startViewTransition()` for element-level transitions

**Barba.js verdict:** Only needed as Firefox MPA fallback. Native API should be default for new projects.

### 2.5 WebGL/Shader Tooling — Three.js TSL Changes Everything

**Three.js r171+ (Sep 2025):** Zero-config WebGPU support with WebGL 2 fallback.

**TSL (Three Shading Language):** Write shaders in JavaScript, auto-compiles to GLSL (WebGL) and WGSL (WebGPU):
```javascript
const timeUniform = uniform(0);
material.colorNode = sin(timeUniform).mul(0.5).add(0.5);
```
**This is a game-changer for AI code generation** — no more GLSL string templates, just JavaScript function composition with full IDE support.

**WebGPU browser support (Nov 2025):** Chrome 113+, Edge 113+, Firefox 139+, Safari 26+. **All major browsers.** 2-10x performance over WebGL.

**OGL:** 29KB minimal WebGL library. Worth documenting as lightweight Three.js alternative for simple shader backgrounds.

**AI shader generation:** ShaderGPT, ChatGL exist but produce basic/error-prone output. Best as starting points for human refinement. TSL is more promising for AI-gen because it's JavaScript, not GLSL strings.

### 2.6 Smooth Scroll: Lenis Confirmed

**Lenis v1.3.18** (March 2025) remains the industry standard. No change needed from our current choice.
- Doesn't break `position: sticky` or IntersectionObserver
- Locomotive Scroll v5 rebuilt on top of Lenis
- GSAP ScrollSmoother is the main alternative (now free)
- **Trig.js** (4KB) — new lightweight CSS-first scroll observer, not a Lenis replacement

## Part III: Backend Implementation Tooling

### 3.1 Playwright Visual Testing (v1.50+)

**Built-in VRT API (`toHaveScreenshot()`):**
- Uses pixelmatch under the hood
- `maxDiffPixels`, `maxDiffPixelRatio`, `threshold` for tolerance
- `mask` locators for dynamic content
- `animations: 'disabled'` for consistent captures
- `fullPage: true` for scrollable pages

**v1.57.0:** Removed deprecated accessibility API, recommends `@axe-core/playwright`. Added Speedboard for test performance analysis.

**Best practice:** Generate baselines in Playwright Docker image for cross-platform consistency.

### 3.2 Visual Regression: Pixel vs. AI Diffing

| Tool | Approach | False Positives | Best For |
|---|---|---|---|
| **Playwright + pixelmatch** | Pixel-level | High (anti-aliasing, rendering) | Basic CI gating |
| **Percy (BrowserStack)** | Perceptual AI + OCR | Low | AI-generated design iterations |
| **Applitools Eyes** | Visual AI (computer vision) | Very low (99.9999% claimed) | Enterprise, zero false positives |
| **BackstopJS** | Pixel comparison | Medium | Self-hosted, configurable |

**For AI-generated designs:** Percy or Applitools understand "visual similarity" rather than pixel-perfect matching — critical when comparing iterations where acceptable rendering variation exists.

**Recommendation:** Start with Playwright native for validation gating. Consider Percy for cross-iteration comparison where pixel-perfect is too strict.

### 3.3 Accessibility Automation

**axe-core 4.11.x:** ~57% WCAG detection with zero false positives. WCAG 2.2 AA rules included. Integrates via `@axe-core/playwright`.

**Pa11y-ci:** 68-73% detection, fast CI execution. Can produce false positives.

**WCAG 2.2 became ISO/IEC 40500:2025** in October 2025.

**Combined strategy:** axe-core (zero false positives, CI gating) + Pa11y-ci (fast, broad coverage). Note: all automated tools catch only 30-40% of issues; manual testing still required.

### 3.4 Design Token Tooling — DTCG Spec Now Stable

**W3C DTCG v2025.10** (October 2025): 13 token types, Display P3/Oklch color spaces, theming, cross-platform output. Backed by Adobe, Amazon, Google, Microsoft, Meta, Figma, etc.

**Style Dictionary v4:** First-class DTCG support (`$value`/`$type`/`$description`), ESM rewrite, browser-compatible, auto-converts color objects to CSS strings, typography to `font` shorthand.

**Terrazzo:** DTCG-native CLI (v0.7.2), generates CSS, Sass, JS/TS, Swift, Tailwind. Claims full DTCG format support.

**Dispersa:** TypeScript DTCG build system with six-layer naming convention, GitHub Actions CI/CD, npm publishing.

**Our pipeline's `lib/tokens.js`** does Brief YAML → CSS custom properties. Could upgrade to use Style Dictionary v4 or Terrazzo for DTCG 2025.10 compliance and multi-platform output if needed.

### 3.5 HTML/CSS Validation

**html-validate:** Actively maintained, extensible presets, plugin architecture, integrates with Prettier/Cypress/Netlify. Already in our pipeline.

**Stylelint for token enforcement:**
- `stylelint-declaration-use-css-custom-properties` — enforce custom property usage over raw values
- **Rhythmguard** plugin — `use-scale` (validates spacing) + `prefer-token` (enforces token usage) with autofix

**New:** ESLint expanding to CSS/JSON/Markdown. `@eslint/mcp` package lets AI models interact with ESLint directly.

### 3.6 Performance Budgets & Core Web Vitals

**CWV 2025-2026:** LCP, INP (replaced FID), CLS. Firefox implemented INP (Oct 2025).

**Animation FPS:** No standard CWV metric covers frame rate. Must use custom `requestAnimationFrame`-based measurement:
```javascript
const fps = await page.evaluate(() => {
  return new Promise(resolve => {
    const frames = [];
    let start = performance.now();
    function measure(now) {
      frames.push(now);
      if (now - start > 2000) {
        resolve(1000 / ((frames[frames.length-1] - frames[0]) / (frames.length - 1)));
      } else requestAnimationFrame(measure);
    }
    requestAnimationFrame(measure);
  });
});
```

**Lighthouse CI:** `lighthouserc.js` with assertions, GitHub Actions via `treosh/lighthouse-ci-action`.

## Part IV: Award-Winning Design Patterns 2025-2026

### 4.1 Dominant Aesthetics

1. **Typography IS the hero** — Oversized display fonts, kinetic lettering, variable font animation on scroll/interaction
2. **Interactive 3D environments** — Objects reacting to scroll/hover/motion, environmental storytelling
3. **Cinematic heroes** — Full-bleed video backgrounds with bold type overlays
4. **Mixed scroll directions** — Vertical + horizontal combined, chapter-like narratives
5. **Glow design** — Dynamic lighting + dark mode, futuristic neon
6. **Noise + chromatic mash-ups** — Neon gradients + grain textures + glitch
7. **Illustrated worlds** — Hand-drawn elements as counter-trend to AI perfection
8. **Exploratory layouts** — Scattered, floating, modular elements for click-and-hover discovery

### 4.2 Key Tech Stacks of Award Winners

**Darkroom Engineering** (created Lenis): Next.js 16, React 19, TypeScript strict, GSAP, Theatre.js, Lenis, WebGL, React Three Fiber. CMS: Contentful/Sanity. Hosting: Vercel.

**2024 SOTY: Igloo Inc** (by Abeto): Immersive 3D + scroll interaction + micro-interactions.

**Codrops frontier (2025):** GSAP (12+ tutorials), Three.js, R3F, WebGPU, real-time dithering shaders, fluid simulations, scroll-driven 3D text, infinite marquees, video projection mapping, 3D audio visualizers. 1,000+ open-source demos.

### 4.3 Emerging CSS Features (2026 Baseline)

| Feature | Impact | Status |
|---|---|---|
| `@starting-style` | Animate from `display: none` | Cross-browser baseline |
| CSS Masonry | Native Pinterest-style layout | Firefox 140, Chrome in dev |
| CSS Gap Decorations | `column-rule`/`row-rule` in grid/flex | Chrome 147 expected |

### 4.4 Typography Specifics

**Popular typefaces (2025-2026):** Graphik (editorial), Aeonik (neo-grotesque), Inter (screen-optimized), Druk (bold condensed), Romie (calligraphy-modern serif).

**Variable fonts are central:** Animate `font-weight`, `font-width` on scroll/interaction. Single file reduces page load.

**Counter-trend:** Imperfect/hand-drawn aesthetics — wobbly outlines, risograph textures, "wrong" spacing as rebellion against AI perfection.

### 4.5 Color & Visual Treatment

- Mesh gradients: layered radial-gradients at different positions. CSS has no native mesh gradient; **WebGL preferred for animated mesh gradients** (GPU-efficient).
- Glassmorphism: `backdrop-filter: blur()` + semi-transparent backgrounds + subtle shadows. Paired with dark mode.
- Grain/texture: micro-textures, soft grain — reducing coldness, signaling human touch.
- New CSS: `oklch(from ...)` relative color, `contrast-color()`, `color-mix()`.

### 4.6 Sound Design & Haptics (Emerging)

- Web Audio API is now a W3C Recommendation
- Award examples: Fornasetti Profumi ASMR Rooms, Into the Storm (interactive documentary), Moooi Sound Landscape
- Spatial audio (3D positional) possible without plugins
- Haptic: Vibration API for PWAs. One site saw 27% increase in form completion.
- All audio must be opt-in per browser autoplay policies.
- 3D audio visualizers (Three.js + Web Audio API) a Codrops 2025 focus.

### 4.7 AI Integration in Front-End

- **Generative UI:** Interface restructures in real-time per user intent — layout, navigation, content adapt (Google, Vercel AI SDK)
- **Machine Experience (MX) Design:** Semantic HTML + structured content for AI consumption. Future-proofing for AI search engines.

## Part VI: Recommended Capability Expansions

### Tier 1 — High Impact, Implement Now

#### 1. Design Constitution Enforcement
**What:** Add strict rules to the generator agent that forbid raw CSS values. All styling must reference design tokens.
**Why:** The Codrops project proved this is the single most effective technique for preventing "AI slop." Without it, LLMs produce "an average of all designs worldwide."
**How:**
- Add rule to `.claude/agents/generator.md`: "You are FORBIDDEN from using raw color, spacing, or typography values. Every value must reference a CSS custom property from tokens.json."
- Add Stylelint with Rhythmguard plugin to validation pipeline: `prefer-token` rule rejects raw values, `use-scale` validates spacing.
- Add `stylelint-declaration-use-css-custom-properties` to enforce `var(--*)` usage.

#### 2. Expand GSAP Plugin Documentation
**What:** Update pattern docs to cover newly-free plugins: SplitText (rewritten), MorphSVG, DrawSVG, Flip, ScrollSmoother, Physics2D, Inertia, ScrambleText.
**Why:** These were previously $99+/year. Now free with no registration. Dramatically expands our animation vocabulary.
**How:**
- Update `patterns/effects/typography-animation.md` with SplitText v3.13: `autoSplit`, `deepSlice`, `Intl.Segmenter`, built-in masking, ARIA attributes.
- Add SVG animation section to patterns covering MorphSVG + DrawSVG.
- Add `patterns/effects/layout-animation.md` covering Flip plugin techniques.
- Update CDN references throughout to include all plugins.

#### 3. CSS Scroll-Driven Animation Patterns
**What:** Document CSS-native scroll animations as the first choice for simple effects, with GSAP ScrollTrigger for complex ones.
**Why:** ~83% browser support, zero JavaScript, compositor-threaded performance.
**How:**
- Add CSS scroll-driven section to `patterns/scroll-sections.md` with `@supports` feature detection.
- Document progressive enhancement: CSS `animation-timeline` base → GSAP ScrollTrigger enhancement.
- Include Firefox polyfill strategy (`flackr/scroll-timeline`).

#### 4. Stylelint Integration in Validation Pipeline
**What:** Add Stylelint to `lib/validate.js` as a validation check.
**Why:** Automated enforcement of design token usage — catches what the Design Constitution rules might miss.
**How:**
- `npm install -D stylelint stylelint-declaration-use-css-custom-properties`
- Configure `.stylelintrc.json` with rules enforcing `var(--*)` usage.
- Add Stylelint pass to validation runner alongside html-validate.

### Tier 2 — Medium Impact, Implement Soon

#### 5. Three.js TSL for Shader Backgrounds
**What:** Document TSL (Three Shading Language) as the preferred shader authoring method, replacing raw GLSL strings.
**Why:** TSL is JavaScript function composition — dramatically easier for AI code generation. Auto-compiles to GLSL (WebGL) and WGSL (WebGPU).
**How:**
- Update `patterns/effects/backgrounds.md` with TSL examples for noise, gradient mesh, particle fields.
- Add TSL-based shader pattern: `material.colorNode = sin(timeUniform).mul(0.5).add(0.5)`.

**Why:** ~90% global support for same-document transitions. Dramatically simpler than Barba.js.
**How:**
- Rewrite page-transitions.md with View Transitions API as primary approach.
- Document `view-transition-name: match-element` for auto-naming.
- Document Barba.js as progressive enhancement for Firefox MPA only.

#### 7. Deployment Pipeline (shipper agent)
**What:** Add Cloudflare Pages deployment to the shipper agent with preview URLs.
**Why:** Preview URLs enable design review workflow — stakeholders see iterations without local setup.
**How:**

- Update `.claude/agents/shipper.md` with deployment steps.
- Each design gets a preview URL for review feedback.

#### 8. Lighthouse CI Integration

**What:** Document Three.js WebGPU renderer as optional performance upgrade.

**How:** Add WebGPU section to background effects patterns. Progressive enhancement: WebGPU → WebGL fallback.

#### 14. Percy/Applitools AI Visual Diffing
**What:** Optional upgrade from pixelmatch to AI-aware visual comparison.
**Why:** AI-generated designs have acceptable variation between iterations that pixel-matching flags as failures.
**How:** Integrate Percy or Applitools as optional validation backend. Keep pixelmatch as default (free, sufficient for most cases).

#### 15. Generative UI Patterns
**What:** Document AI-adaptive front-end patterns for designs that personalize per user.
**Why:** Emerging trend. Google and Vercel both shipping Generative UI primitives.
**How:** Future phase. Add when the pipeline handles multi-page/dynamic sites.
